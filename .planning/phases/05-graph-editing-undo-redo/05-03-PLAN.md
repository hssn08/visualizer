  <action>
Create `src/hooks/useUndoRedo.ts`:
- Export `useUndoRedo()` hook
- Inside, use `useEffect` to attach a global `keydown` listener on `window`
- Handler logic per RESEARCH.md Pattern 5:
  - If `(e.ctrlKey || e.metaKey) && e.key === 'z'`:
    - `e.preventDefault()` (prevent browser undo in text inputs)
    - If `e.shiftKey`: call `useAppStore.temporal.getState().redo()`
    - Else: call `useAppStore.temporal.getState().undo()`
- Return cleanup function that removes the listener
- The hook takes no parameters and returns nothing -- it's a side-effect-only hook

Create `src/hooks/__tests__/useUndoRedo.test.ts`:
- Mock `useAppStore.temporal.getState` to return `{ undo: vi.fn(), redo: vi.fn() }`
- Use `renderHook` from `@testing-library/react` to mount the hook
- Dispatch `KeyboardEvent('keydown', { key: 'z', ctrlKey: true })` on window
- Verify `undo` was called
- Dispatch with `shiftKey: true` additionally
- Verify `redo` was called
- Test metaKey variants
- Test that plain 'z' does not trigger undo
- Test cleanup on unmount

Update `src/App.tsx`:
- Import and call `useUndoRedo()` at the top of the App component body
- This ensures the keyboard listener is active app-wide

Update `src/components/canvas/FlowCanvas.tsx`:
- Add `onNodeDragStart` and `onNodeDragStop` callbacks per RESEARCH.md Pattern 3:
  ```typescript
  const onNodeDragStart = useCallback(() => {
    useAppStore.temporal.getState().pause();
  }, []);

  const onNodeDragStop = useCallback(() => {
    useAppStore.temporal.getState().resume();
  }, []);
  ```
- Wire both as props on the `<ReactFlow>` component:
  ```
  onNodeDragStart={onNodeDragStart}
  onNodeDragStop={onNodeDragStop}
  ```

Note per RESEARCH.md Pitfall 2: React Flow fires onNodeDragStop even when a node is clicked without being dragged. This is harmless because Zundo's equality check compares partialized state (nodes + edges). If nothing changed during the pause window, no undo entry is created. No need for additional position-comparison logic.

- Test: Add a node via addNode. Call `undo()`. Verify the node was removed.
- Test: Verify that multiple changes during pause produce exactly one undo step (undo once reverts all changes made during pause).
