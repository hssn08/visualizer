- Implement drop handling using the pointer-event DnD pattern from RESEARCH.md:
  - Use `onDragOver` (preventDefault to allow drop) and `onDrop` on the ReactFlow wrapper
  - On drop: read the template type from the dataTransfer, call `screenToFlowPosition({x: event.clientX, y: event.clientY})`, generate a unique node ID using `generateNodeId`, create a new Node with `type: 'step'`, the template's `defaultStep` as `data.step`, `data.label` as template label, `data.isFirstNode: false`, and add via `addNode`
- Alternative simpler approach: Use HTML5 drag dataTransfer.setData in PaletteItem's onDragStart, and onDrop in FlowCanvas reads it. This is simpler than the full pointer-event pattern and works for this use case.

Update `src/App.tsx`:
- Replace the `{/* Sidebar placeholder - Phase 5 */}` comment with `<NodePalette />`
- Import `NodePalette` from `@/components/palette/NodePalette`
- Layout: NodePalette appears before the `flex-1` canvas div in the flex row

The DnD pattern:
- PaletteItem: `onDragStart={(e) => { e.dataTransfer.setData('application/reactflow-template', template.type); e.dataTransfer.effectAllowed = 'move'; }}`  with `draggable`
- FlowCanvas wrapper: `onDragOver={(e) => { e.preventDefault(); e.dataTransfer.dropEffect = 'move'; }}`
- FlowCanvas: `onDrop` handler reads template type, creates node, adds to store

Note: React Flow's official DnD example uses pointer events, but the HTML5 drag API with dataTransfer is actually simpler and well-supported for sidebar-to-canvas DnD. Both work. Use whichever is cleaner. The key requirement is that `screenToFlowPosition` converts drop coordinates to flow coordinates.
