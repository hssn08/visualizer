  const [deleteConfirm, setDeleteConfirm] = useState<{

- Implement `onBeforeDelete` callback:
  - If `nodes.length === 0`: return `true` (edge-only deletion needs no confirmation)
  - Otherwise: return a `new Promise<boolean>` that stores the resolve function in `deleteConfirm` state
- Implement `onDelete` callback:
  - When nodes/edges are actually deleted (after confirmation), call `onEdgesDelete` for each deleted edge to sync step data
  - For edges connected to deleted nodes (cascading deletion), also sync step data cleanup
- Wire `onBeforeDelete` and `onDelete` props on ReactFlow component
- Set `deleteKeyCode={['Backspace', 'Delete']}` on ReactFlow to enable keyboard deletion (UI-03)
- Render AlertDialog JSX inside the FlowCanvas return, per RESEARCH.md Pattern 4:
  - `<AlertDialog open={!!deleteConfirm}>`
  - Title: "Delete Node?"
  - Description: "This will remove [node id] and its connected edges. This action cannot be undone."
  - Cancel button: resolves false, clears deleteConfirm
  - Delete button: resolves true, clears deleteConfirm (use destructive variant)
  - `onOpenChange` handler: if closed without action, resolve false

Note on edge-only deletion: When a user selects and deletes just an edge (not a node), `onBeforeDelete` receives `nodes: []`. Return true immediately so edges can be deleted without confirmation. The `onDelete` callback still fires and should sync step data via `onEdgesDelete`.
