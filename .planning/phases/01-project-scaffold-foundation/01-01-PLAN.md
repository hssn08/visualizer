<task type="auto">
  <name>Task 1: Scaffold Vite project and install all dependencies</name>
  <files>package.json, vite.config.ts, tsconfig.json, tsconfig.app.json, tsconfig.node.json, index.html</files>
  <action>
1. Create the Vite project in the CURRENT directory (it's already named "visualizer" and is an empty git repo with only .planning/). Use `npm create vite@latest . -- --template react-ts` to scaffold in-place. If prompted about existing files, proceed (the template will not overwrite .planning/).

2. Install all dependencies in one batch:
   ```
   npm install @xyflow/react zustand zundo lucide-react clsx tailwind-merge class-variance-authority tw-animate-css
   npm install -D @tailwindcss/vite tailwindcss vitest @testing-library/react @testing-library/dom jsdom @types/react @types/react-dom
   ```

3. Configure vite.config.ts with the React plugin, Tailwind v4 plugin, and @ path alias:
   ```typescript
   import path from "path";
   import tailwindcss from "@tailwindcss/vite";
   import react from "@vitejs/plugin-react";
   import { defineConfig } from "vite";

   export default defineConfig({
     plugins: [react(), tailwindcss()],
     resolve: {
       alias: {
         "@": path.resolve(__dirname, "./src"),
       },
     },
   });
   ```

4. Add path aliases to BOTH tsconfig.json AND tsconfig.app.json:
   - In tsconfig.json: add `"baseUrl": "."` and `"paths": { "@/*": ["./src/*"] }` to compilerOptions
   - In tsconfig.app.json: add `"baseUrl": "."` and `"paths": { "@/*": ["./src/*"] }` to compilerOptions

5. Create vitest.config.ts at the project root:
   ```typescript
   import { defineConfig } from 'vitest/config';
   import react from '@vitejs/plugin-react';
   import path from 'path';

   export default defineConfig({
     plugins: [react()],
     test: {
       environment: 'jsdom',
       globals: true,
       setupFiles: [],
     },
     resolve: {
       alias: {
         '@': path.resolve(__dirname, './src'),
       },
     },
   });
   ```

6. Remove the default Vite boilerplate files: delete src/App.css, src/assets/ (if created), and the default counter code from App.tsx. Keep src/main.tsx and src/App.tsx but they will be overwritten in the next task and plan.

IMPORTANT: Do NOT use `tailwindcss-animate` (deprecated for v4). Use `tw-animate-css` instead. Do NOT create a `tailwind.config.js` file -- Tailwind v4 uses CSS-first config.
  </action>
  <verify>
    <automated>cd /root/visualizer && npx tsc --noEmit && npx vite build 2>&1 | tail -5</automated>
  </verify>
  <done>Vite project builds without errors. All dependencies in package.json. Path alias @ resolves. Vitest config exists with jsdom environment.</done>
</task>

<task type="auto">
  <name>Task 2: Configure Tailwind v4 CSS, shadcn/ui, and initial components</name>
  <files>src/index.css, src/lib/utils.ts, components.json, src/components/ui/button.tsx, src/App.tsx, src/main.tsx</files>
  <action>
1. Create src/lib/utils.ts with the cn() utility:
   ```typescript
   import { clsx, type ClassValue } from "clsx";
   import { twMerge } from "tailwind-merge";

   export function cn(...inputs: ClassValue[]) {
     return twMerge(clsx(inputs));
   }
   ```

2. Create src/index.css with the CORRECT import order (this is critical -- React Flow CSS MUST come before Tailwind):
   ```css
   /* 1. React Flow base styles FIRST (before Tailwind resets them) */
   @import "@xyflow/react/dist/style.css";

   /* 2. Tailwind CSS */
   @import "tailwindcss";

   /* 3. Animation utilities for shadcn */
   @import "tw-animate-css";

   /* 4. Tailwind v4 dark mode variant */
   @custom-variant dark (&:is(.dark *));
   ```
   Note: The shadcn theme variables (@theme inline block with color mappings, @layer base styles) will be added by `npx shadcn@latest init`. If the init command does not add them automatically, add the full theme block manually following the shadcn/ui Vite installation guide. The theme must include CSS variable mappings like `--color-background: var(--background)` etc.

3. Ensure src/main.tsx imports index.css:
   ```typescript
   import React from 'react';
   import ReactDOM from 'react-dom/client';
   import App from './App';
   import './index.css';

   ReactDOM.createRoot(document.getElementById('root')!).render(
     <React.StrictMode>
       <App />
     </React.StrictMode>,
   );
   ```

4. Run `npx shadcn@latest init` to initialize shadcn/ui. When prompted:
   - Style: default (or new-york)
   - Base color: neutral
   - CSS variables: yes
   - React Server Components: NO (this is a Vite SPA, not Next.js)
   - If it asks about tailwind.config: there is none, use CSS config

   If the CLI has issues, create components.json manually:
   ```json
   {
     "$schema": "https://ui.shadcn.com/schema.json",
     "style": "default",
     "rsc": false,
     "tsx": true,
     "tailwind": {
       "config": "",
       "css": "src/index.css",
       "baseColor": "neutral",
       "cssVariables": true,
       "prefix": ""
     },
     "aliases": {
       "components": "@/components",
       "utils": "@/lib/utils",
       "ui": "@/components/ui",
       "lib": "@/lib",
       "hooks": "@/hooks"
     },
     "iconLibrary": "lucide"
   }
   ```

5. Add the shadcn Button component: `npx shadcn@latest add button`

6. Create a minimal src/App.tsx that proves the stack works:
   ```typescript
   import { Button } from '@/components/ui/button';

   export default function App() {
     return (
       <div className="h-screen w-screen flex items-center justify-center bg-background text-foreground">
         <div className="text-center space-y-4">
           <h1 className="text-2xl font-bold">Flow Editor</h1>
           <Button>Ready</Button>
         </div>
       </div>
     );
   }
   ```

7. Verify that after `npm run dev`, the page shows "Flow Editor" heading with a styled Button. The button should have shadcn/ui styling (not unstyled HTML).

IMPORTANT: If the CSS @import for React Flow does not resolve through Vite's CSS pipeline, fall back to importing it in main.tsx as a JS import: `import '@xyflow/react/dist/style.css';` BEFORE the `import './index.css'` line.

IMPORTANT: Ensure the React Flow CSS import is BEFORE the Tailwind import. Tailwind's Preflight reset will strip React Flow's base styles if loaded after.
  </action>
