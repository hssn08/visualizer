  <name>Task 1: flowToJson reverse transform with round-trip tests</name>
  <files>src/lib/flowToJson.ts, src/lib/__tests__/flowToJson.test.ts, src/lib/__tests__/roundTrip.test.ts</files>
  <behavior>
    - flowToJson returns a Record with wrapperFields restored at top level
    - flowToJson places rebuilt steps at metadata.stepsKey
    - Each step in output contains all original fields from node.data.step
    - Connection field "next" is updated from edges with edgeType "next"
    - Connection field "conditions[i].next" is updated from edges with edgeType "condition" and matching conditionIndex
    - Connection field "timeout_next" is updated from edges with edgeType "timeout"
    - Connection field "no_match_next" is updated from edges with edgeType "no_match"
    - Connection field "intent_detector_routes" is rebuilt from edges with edgeType "intent"
    - If no edge exists for "next", the field is removed (not set to undefined/null in JSON output)
    - Round-trip: jsonToFlow -> flowToJson produces JSON with same keys and values as input
    - Round-trip preserves wrapper fields exactly (flow_name, version, voice_settings)
    - Round-trip preserves non-visual step fields (max_clarification_retries, criticalstep, action, disposition)
  </behavior>
  <action>
1. Create `src/lib/flowToJson.ts`:
   - Export `flowToJson(nodes: Node[], edges: Edge[], metadata: JsonMetadata): Record<string, unknown>`
   - Implementation per research Pattern 4:
     a. Start with spread of metadata.wrapperFields
     b. Build steps object from nodes: for each node, clone node.data.step via spread
     c. For each step, call internal `updateConnectionFields(step, nodeId, edges)`:
        - Find edges with `source === nodeId`
        - Set step.next from edgeType "next" edge target (delete if no such edge)
        - Update conditions[i].next from edgeType "condition" edges matching conditionIndex
        - Set timeout_next from edgeType "timeout" (delete if absent)
        - Set no_match_next from edgeType "no_match" (delete if absent)
        - Rebuild intent_detector_routes from edgeType "intent" edges (delete if no intent edges)
     d. Place steps at result[metadata.stepsKey]
   - IMPORTANT: Use `delete step.fieldName` for absent connections rather than setting to undefined. This ensures `JSON.stringify` does not emit `"field": null`.

2. Write `src/lib/__tests__/flowToJson.test.ts`:
   - Test basic reconstruction: flowToJson with single node (next edge) produces correct JSON
   - Test connection field updates: modify an edge target, verify exported JSON reflects change
   - Test missing connection: remove a "next" edge, verify "next" field deleted from step
   - Test conditions array: conditions[0].next updated from edge target
   - Test intent routes: rebuilt from intent edges

3. Write `src/lib/__tests__/roundTrip.test.ts`:
   - Load sampleFlow.json fixture
   - jsonToFlow(sample) -> flowToJson(nodes, edges, metadata)
   - Deep compare input and output (use JSON.parse(JSON.stringify()) to normalize)
   - Specifically verify: flow_name, version, voice_settings preserved
   - Specifically verify: max_clarification_retries, criticalstep preserved in step data
   - Specifically verify: all condition objects preserved with their non-next fields

4. RED -> GREEN -> commit pattern.
