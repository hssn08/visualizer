  <files>src/lib/types.ts, src/lib/detectSteps.ts, src/lib/edgeExtractors.ts, src/lib/__tests__/fixtures/sampleFlow.json, src/lib/__tests__/detectSteps.test.ts, src/lib/__tests__/edgeExtractors.test.ts</files>
  <behavior>
    - detectStepsContainer finds "steps" key in sample JSON (top-level object-of-objects with linking fields)
    - detectStepsContainer throws descriptive error when given JSON with no valid steps container
    - detectStepsContainer picks the key with the highest linking field count when multiple candidates exist
    - detectStepsContainer works for single-level nested keys (e.g., object at top level whose children are objects with linking fields)
    - detectStepsContainer requires at least 2 step objects to avoid false positives
    - extractEdgesFromStep extracts a "next" edge (edgeType: "next") from step.next string
    - extractEdgesFromStep extracts condition edges (edgeType: "condition") from step.conditions array, with label from condition_description
    - extractEdgesFromStep extracts timeout edge (edgeType: "timeout") from step.timeout_next
    - extractEdgesFromStep extracts no_match edge (edgeType: "no_match") from step.no_match_next
    - extractEdgesFromStep extracts intent edges (edgeType: "intent") from step.intent_detector_routes object
    - extractEdgesFromStep produces unique edge IDs even when multiple edges target the same node
    - extractEdgesFromStep returns empty array for steps with no linking fields
  </behavior>
  <action>
1. Create test fixture `src/lib/__tests__/fixtures/sampleFlow.json` with the Medicare call flow sample from the research document. Include at minimum: greeting (next + timeout_next), verify_identity (conditions + no_match_next + intent_detector_routes), transfer_agent (no next), plus wrapper fields (flow_name, version, voice_settings).

2. Create `src/lib/types.ts` with these types:
   - `FlowTransformResult`: `{ nodes: Node[], edges: Edge[], metadata: JsonMetadata }`
   - `JsonMetadata`: `{ stepsKey: string, wrapperFields: Record<string, unknown> }`
   - `StepContainerResult`: `{ stepsKey: string, steps: Record<string, Record<string, unknown>> }`
   - Import Node, Edge from `@xyflow/react`

3. Create `src/lib/detectSteps.ts`:
   - Export `detectStepsContainer(json: Record<string, unknown>): StepContainerResult`
   - LINKING_FIELDS constant: `['next', 'conditions', 'timeout_next', 'no_match_next', 'intent_detector_routes']`
   - Algorithm: iterate top-level keys, for each value that is a plain object, count child objects that have linking fields. Pick the key with highest linking field count. Require at least 2 step objects.
   - Export `isPlainObject` helper (used by edgeExtractors too).
   - Throw descriptive Error if no container found.

4. Create `src/lib/edgeExtractors.ts`:
   - Export `extractEdgesFromStep(stepKey: string, step: Record<string, unknown>): Edge[]`
   - Handle all 5 edge types per research Pattern 3. Each edge gets:
     - Unique ID format: `${stepKey}->${type}->${target}` (with index for conditions, intent name for intents)
     - `sourceHandle` matching the type (next, condition-N, timeout, no_match, intent-NAME)
     - `data.edgeType` for Phase 3 styling
     - `label` where applicable (condition_description, "Timeout", "No Match", intent name)
   - Use `isPlainObject` from detectSteps for intent_detector_routes check.

5. Write tests FIRST (RED), then implement (GREEN). Commit test files before implementation.

    - Each node has data.step containing the FULL original step object (spread copy)
    - Node positions use grid layout (4 columns, 300px X gap, 200px Y gap)
    - metadata.stepsKey matches the detected key (e.g., "steps")
    - metadata.wrapperFields contains all non-step keys (flow_name, version, voice_settings)
    - All edges from all steps are collected (next, conditions, timeout, no_match, intents)
    - Edges referencing nonexistent target step keys are filtered out (no dangling edges)
  </behavior>
  <action>
1. Create `src/lib/jsonToFlow.ts`:
   - Import `detectStepsContainer` from `./detectSteps`
   - Import `extractEdgesFromStep` from `./edgeExtractors`
   - Import types from `./types`
   - Export `jsonToFlow(rawJson: Record<string, unknown>): FlowTransformResult`
   - Implementation:
     a. Call `detectStepsContainer(rawJson)` to get stepsKey and steps
     b. Build wrapperFields: all top-level keys except stepsKey
     c. Convert steps to nodes via `stepsToNodes` (internal function):

        - position = grid layout (index % 4 * 300, floor(index / 4) * 200)

   - Test dangling edge filtering (add a step with next pointing to nonexistent key)
